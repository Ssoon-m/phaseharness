# Task Creation Prompt

Use this prompt when a requirement and implementation plan already exist and must be converted into harness task files.

## Required Outputs

Create:

- `tasks/index.json`
- `tasks/<task-dir>/index.json`
- `tasks/<task-dir>/artifacts/02-context.md`
- `tasks/<task-dir>/artifacts/03-plan.md`
- `tasks/<task-dir>/phase0.md`
- one `tasks/<task-dir>/phase<N>.md` file per implementation phase

Preserve `tasks/<task-dir>/artifacts/01-clarify.md`; it is written by
main-session clarify before this prompt is used.

## Task Index Contract

Append a new entry to `tasks/index.json`.

```json
{
  "tasks": [
    {
      "id": 1,
      "name": "example",
      "dir": "1-example",
      "status": "pending",
      "created_at": "2026-04-26T12:00:00+0900"
    }
  ]
}
```

## Task Phase Index Contract

Create `tasks/<task-dir>/index.json`.

```json
{
  "project": "<project-name>",
  "task": "<task-name>",
  "prompt": "<original requirement summary>",
  "status": "pending",
  "totalPhases": 2,
  "created_at": "2026-04-26T12:00:00+0900",
  "done_when": [
    "<observable end condition>"
  ],
  "max_attempts": 2,
  "artifacts": {
    "clarify": "artifacts/01-clarify.md",
    "context": "artifacts/02-context.md",
    "plan": "artifacts/03-plan.md",
    "generate": "artifacts/04-generate.md",
    "evaluate": "artifacts/05-evaluate.md"
  },
  "phases": [
    {
      "phase": 0,
      "name": "docs",
      "status": "pending",
      "max_attempts": 2,
      "commit_message": "docs: update project documentation"
    },
    {
      "phase": 1,
      "name": "implementation",
      "status": "pending",
      "max_attempts": 2,
      "commit_message": "feat: implement requested behavior"
    }
  ],
  "evaluation": {
    "status": "pending",
    "attempts": 0,
    "max_attempts": 2
  }
}
```

## Phase Rules

- Phase 0 must update docs when docs need to change.
- Do not overwrite `artifacts/01-clarify.md`.
- Phase files must be self-contained enough for an independent session.
- Phase files must include concrete paths, scope, and acceptance criteria.
- Acceptance criteria must be runnable commands.
- Every task must state `done_when` clearly enough for evaluation to pass or fail it.
- Every phase must have a `max_attempts` value. Use the orchestrator value unless
  there is a concrete reason to lower it.
- Every phase should include a concise `commit_message` that describes the user
  visible work. Do not mention phase numbers or phaseloop internals.
- `docs-diff.md` is generated by `scripts/gen-docs-diff.py`; do not write it manually.
- `artifacts/04-generate.md` is appended by phase execution.
- `artifacts/05-evaluate.md` is written by the evaluation step.

## Phase File Template

````markdown
# Phase <N>: <name>

## Context
- Requirement source:
- Relevant docs:
- Relevant files:

## Work
- Concrete implementation steps.

## Acceptance Criteria
```bash
<command>
```

## Status Update
If all acceptance criteria pass, update `tasks/<task-dir>/index.json` so phase <N> has `"status": "completed"`.
If the phase cannot be completed after reasonable fixes, set `"status": "error"` and add `"error_message"`.
````
